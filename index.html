Thinking Like a Programmer

Procedural Thinking: Create perfectly clear and unambiguous instructions for a computer to follow. (Because computers are stupid)

Systems Thinking: Break a big problem down into smaller, more manageable components

----->Generate Theories
----->Test Theories
----->Resolve the Problem

Basics of Web and HTML

The web is: A network of servers that communicate with one another, responding to HTTP requests and returning appropriate HTML doc and displays on the users screen.
Computers are stupid. Computers interpet instructions literally, always. Very rigid and onforgiving.
Programmers cannot remember everything: Google is your friend. The key is knowing the questions to ask.
HTML: Hyper Text Markup Language. Makes up the majority of content on the web. Made up of:
-----> Text content (what you see),
-----> Markup (what it looks like),
-----> References to other documents, images, videos, et al and Links (to other pages)
-----> HTML markup is comprised of tags, content (text) and elements
Web Applications: Software that runs in a web browser.
Basic HTML vocabluary: Tags and Elements
-----> An HTML doc is comprised of elements.
-----> Elements contain tags and text.

Lesson 3

Adding CSS Style to HTML Structure

Definitions:
- CSS: Cascading Style Sheets. This is code written to control the style of HTML elements. Think of it as a "search and replace" tool. You ID a class or a tag and then define a rule for what you want to do with it, visually. The 'cascading' refers to how the rules nest within one another, most specific rule appplies.
- CSS Inheritance : Properties are applies to a specified element AND its descendeants
- CSS Selector: Allows you to target specfic HTML elements and apply style to them.


3 Ways to include CSS Styling in a WebPage

* Write CSS in the head of your HTML
* Link your HTML to a separate CSS file (Preferred)
--- One file = HTML
--- One file = CSS
--- Add head element to top of HTML
--- Add link tag inside the head element
--- Add attributes to link
* Write your style inline with your HTML. (Bad juju; results in duplicate code)


Box Model

HTML elements are boxes and boxes are comprise of four components:
* Content. The text or image at the center of the box.
* Padding. The clear area around the content.
* Border. Outlines the content+padding.
* Margin. Transparent area around the border.
--- Size of an element = Content+padding+border.
--- Does not include margin.
Getting the size of a box right can be a chore.
* Two techniques can help:
--- Set size as a % rather than hard set pixels
--- Set the box-sizing attribute to border-box for each element

Browser Specific Prefixes: Boxes will not render exactly the same in each browser. Browser specific prefizes can make it somewhat agnostic.
* webkit-box-sizing: border-box;
* moz-box-sizing: border-box;
* ms-box-sizing: border-box;
* box-sizing: border-box;


Box positioning

Divs are block elements and thus take up the entire width of the screen.
--- Adding rule display:flex; to CSS wil over ride this behavior and allow divs to appear next to one another.
Code, Test & Refine: Basic developer practice of continuous improvement. Do, inspect, adapt.
--- Look for the natural boxes
--- Look for repeated styles and semantic elements
--- Write HTML
--- Apply styles (CSS) from biggest to smallest
--- Inspect and change. Go over HTML and CSS, inspecting results and modifying code until you are satisfied.

Introduction to Serious Programming

Computer: A universal machine that can be programmed to perform any computation
Program: A very precise sequence of steps that instruct a computer what to do
Programming Language: A formal, constructed language designed to communicate instructions to a computer
Interpreter: A computer program that directly performs instructions written in a programing language without previously compiling them into machine language
Python: A general purpose, high level programming language. An interpreter.
Python Expressions: A combination of values, variables and operators.
Grammar: A set of rules for strings in a formal language
